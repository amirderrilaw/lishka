המשתמש הפעיל נרשם כמבצע הפעולות ביומן הפעילות. מחליפים משתמש בלחיצה על העיגול בפינת המסך.

חיבור לענן גוגל (רב־משתמשים)
לא מחובר — המערכת פועלת במצב מקומי
איך מפיקים מזהה לקוח? (מדריך של 10 דקות, פעם אחת למשרד)
נכנסים אל console.cloud.google.com עם חשבון הגוגל של המשרד ויוצרים פרויקט חדש (New Project) בשם ״לשכה״.
בתפריט: APIs & Services ← Library — מחפשים ומפעילים (Enable) שניים: Google Drive API ו־Google Calendar API.
OAuth consent screen ← בוחרים External ← ממלאים שם ״לשכה״ ואימייל ← ב־Test users מוסיפים את כתובות הג׳ימייל של כל עורכי הדין במשרד.
Credentials ← Create Credentials ← OAuth client ID ← סוג: Web application.
תחת Authorized JavaScript origins מוסיפים את כתובת האתר שבה המערכת מתארחת (השדה למעלה), למשל:
מעתיקים את ה־Client ID שנוצר, מדביקים בשדה למעלה ולוחצים ״שמירת הגדרות״ — ומתחברים.
איפה מעלים את הקובץ לאינטרנט? (נדרש להתחברות גוגל)
הדרך הפשוטה: GitHub Pages (חינם) — נרשמים ב־github.com, יוצרים מאגר (Repository) ציבורי חדש, מעלים את קובץ המערכת בכפתור ״Add file ← Upload files״, ואז ב־Settings ← Pages בוחרים Branch: main ← Save. בתוך דקות תתקבל כתובת קבועה. חלופות: Netlify, Cloudflare Pages, או אחסון האתר הקיים של המשרד. חשוב: הקובץ עצמו אינו מכיל תיקים — כל הנתונים נשמרים בחשבונות הגוגל, ולכן מותר לאחסן אותו בכל מקום.
אזור מסוכן: מחיקת כל הנתונים במערכת לצמיתות (תיקים, מסמכים, הגדרות).
מדריך מהיר
ניהול תיקים
מוסיפים תיק בכפתור ״תיק חדש״. כל תא בטבלה ניתן לעריכה בלחיצה אחת — שם, סוג, עדכון, משימה, מועד, אחראי וסטטוס. לחיצה על שם התיק פותחת את כרטיס התיק המלא עם הערות, מסמכים ויומן פעילות. מיון — בלחיצה על כותרת עמודה; סינון — בצ׳יפים ובתיבות שמעל הטבלה.
מסמכים: העלאה, תפריט אפשרויות, תמלול
גוררים קבצים ללשונית ״מסמכים״ (או מעלים בכפתור). לחיצה אחת על מסמך פותחת תפריט אפשרויות: פתיחה, תמלול לצד המסמך, שמירה כ־PDF, הדפסה, הורדה, שיוך לתיק ועוד. בתצוגת המסמך אפשר לפתוח את חלונית התמלול — הטקסט נשמר אוטומטית לצד הקובץ; במסמכי PDF יש חילוץ טקסט אוטומטי (דורש חיבור לאינטרנט חד־פעמי). בקובצי אודיו יש כפתורי מהירות ודילוג — נוח לתמלול הקלטות.
שיתוף עדכונים בדוא״ל
בלשונית ״שיתוף ועדכונים״ מסמנים תיקים, המערכת מנסחת הודעה מסודרת, ואפשר: לפתוח בתוכנת הדוא״ל, לפתוח ב־Gmail, להעתיק, או להדפיס דוח. אם ההודעה ארוכה — משתמשים בהעתקה ומדביקים בגוף המייל.
עבודה משותפת של כמה עורכי דין
שולחים לעמית את קובץ המערכת (הנתיב מוצג בלשונית השיתוף) — אצלו נפתח עותק עצמאי. לסנכרון: צד אחד מבצע ייצוא, שולח את קובץ הגיבוי במייל, והצד השני מבצע ייבוא ומיזוג — נתונים חדשים גוברים על ישנים, כלום לא הולך לאיבוד. מומלץ לסנכרן בסוף יום עבודה.
איפה הנתונים נשמרים? גיבוי ואבטחה
הכול נשמר מקומית במחשב זה, בתוך הדפדפן — שום מידע לא נשלח לאינטרנט, מה שמתאים לחיסיון עו״ד–לקוח. חשוב להבין: ניקוי נתוני אתרים בדפדפן ימחק גם את המערכת, ולכן מומלץ לייצא גיבוי מלא מדי יום (לוח הבקרה ← ״גיבוי מלא עכשיו״) ולשמור בתיקייה קבועה או בענן המשרדי.
